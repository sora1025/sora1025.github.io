什么是Node.js

JS是脚本语言，脚本语言都需要一个解析器才能运行。对于写在HTML页面里的JS，浏览器充当了解析器的角色，而对于需要独立运行的JS，NodeJS就是一个解析器。

每一种解析器都是一个运行环境。所以Node.js不是一种新的语言，而是运行在Node环境下的JavaScript代码。运行环境允许JS使用运行环境提供的内置对象和方法做一些事情。例如在浏览器中JS的用途是操作DOM，浏览器就提供了document之类的内置对象。而运行在NodeJS中的JS的用途是操作磁盘文件或搭建HTTP服务器。基于此NodeJS就相应提供fs、http等内置对象。

服务端渲染和客户端渲染

最近写node，感觉自己都没有写过HTML，就直接在前端渲染了，噢噢，还了解了一下EJS。。总之，我其实有一点混，说好的前后端分离呢？然后问了一下71，才知道了服务端渲染和客户端渲染这两个概念。

什么是服务端渲染和客户端渲染

互联网早期，用户使用浏览器浏览的都是一些没有复杂逻辑、简单的页面。这些页面都是在后端将HTML拼接好再返回完整的HTML文件给前端，浏览器拿到就可以直接解析展示了，这就是所谓的服务端渲染。当AJAX兴起时，业界就开始推崇前后端分离的开发模式，即后端不提供完整的HTML页面，而是提供一些API使得前端可以获取到JSON数据，前端拿到JSON数据后再在前端进行HTML页面的拼接，然后展示在浏览器上，这就是所谓的客户端渲染。

两者本质的区别是什么

最主要的区别是谁来完成HTML文件的完整拼接

服务端渲染的优缺点

优点
  1. 前端耗时少。因为后端拼接完了HTML，浏览器只需直接渲染出来。
  2. 你有利于SEO。因为在后端有完整的HTML页面，所以爬虫更容易爬取获得信息。
  3. 无需占用客户端资源。即解析模板的工作完全交由后端来做，客户端只要解析标准的HTML页面即可，这样对客户端的资源占用更少，尤其是移动端，也可以更省电。
  4. 后端生成静态化文件。即生成缓存片段，这样就可以减少数据库查询浪费的时间，且对于数据变化不大的页面非常高效。
    缺点
  5. 不利于前后端分离，开发效率低。
  6. 占用服务器端资源

使用服务端渲染还是客户端渲染？

上面讲了各自的区别和优缺点，具体使用哪种渲染方式要看业务场景。比如企业级网站，主要功能是展示而没有复杂的交互，并且需要良好的SEO。就需要使用服务端渲染。类似后台管理界面，交互性比较强，不需要SEO的考虑，那么就可以使用客户端渲染。

另外，具体使用何种渲染方法并不是绝对的，比如现在一些网站采用了首屏服务器端渲染，即对于用户最开始打开的那个页面采用的是服务器端渲染，这样就保证了渲染速度，而其他的页面采用客户端渲染，这样就完成了前后端分离。

Express中的一些方法

path.resolve()路径解析

参考path.resolve()理解

path.resolve() 方法会把一个路径或路径片段的序列解析为一个绝对路径。
给定的路径的序列是从右往左被处理的，后面每个 path 被依次解析，直到构造完成一个绝对路径。 例如，给定的路径片段的序列为：/foo、/bar、baz，则调用 path.resolve(‘/foo’, ‘/bar’, ‘baz’) 会返回 /bar/baz。

path.resolve([…paths])里的每个参数都类似在当前目录进行cd操作，从左到右执行，返回的是最后的当前目录。

最后一个例子同理，../表示上一级目录,从左到右’static_files/png/‘的上一级目录是’/static_files‘，再和后面的路径进行拼接，就得到了最终的路径。

Plus，path.join() 方法名是连接路径，作用是连接任意多个路径字符串。

use方法和get方法

参考nodejs的express使用介绍

最近在跟着写例子的时候，会看到有时用的是app.use，有时用的是app.get，而且感觉写法也很相近，一时不知道两者的区别是什么。看了上面的博客，才算是茅塞顿开。

app.use我知道其中的一个用途是注册中间件。

use方法内部可以对访问路径进行判断，据此就能实现简单的路由，根据不同的请求网址，返回不同的网页内容。

上面判断网址的方法是写在回调函数内部的，use方法还允许将第一个参数作为请求网址。这代表，只有请求路径匹配这个参数，后面的中间件才会生效。这样写更加清晰方便。

get方法相当于use方法的别名，同时加了一些限制。表示只有请求方式是GET的HTTP请求才能通过此中间件。由于get方法的回调函数没有调用next()方法，因此只要有一个中间件被调用了，后面的中间件就不会被调用了。

除了GET方法之外，Express还提供POST,PUT,DELETE方法，即HTTP动词都是Express的方法。

还有一个all方法，表示所有请求必须通过此中间件，参数中的”*”表示对所有路径有效。
