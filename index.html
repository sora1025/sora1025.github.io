作用域与作用域链

Posted on 2018-09-23

（一）
函数是一种特殊的对象，函数类对象，只要是对象就有属性。属性分可访问的和不可访问的。
不可访问的属性仅供JavaScript引擎存取，[[scope]]就是其中一个
test.[[scope]] 指的就是作用域，其中存储了执行期上下文的集合，这个集合呈链式连接，我们把这种链式连接叫做作用域链

作用域因为函数的产生而产生,函数执行时产生执行期上下文，被放在作用域链的最顶端

（二）
1.
function a(){
}
var glob=100;
a();
a函数被定义时 a.[[scope]] —> 0:GO
a函数执行时 a.[[scope]] —->0:AO
1:GO
AO放在作用域链的顶端

2.
function a(){
function b(){
var b=234;
}
var a=123;
b();
}
var glob=100;
a();

b defined [[scope]] —-> 0:aAO 对a的AO环境下产生，引用a的AO
1:GO
b doing [[scope]] —-> 0:bAO b函数执行，产生执行期上下文，置于作用域顶端
—-> 1:aAO
—-> 2:GO

预编译

Posted on 2018-09-13

JS语言特点：①单线程
②解释性语言（解释一行执行一行）
JS运行三步：①语法分析 ②预编译 ③解释执行

将函数声明提升到逻辑的最前面：函数声明整体提升
变量声明提升

1.imply global 暗示全局变量：任何变量，如果变量未经声明就赋值，此变量就为全局对象所有
即console.log(a)会报错
而a=123;
console.log(a);不报错，显示a的值
2.一切声明的全局变量，全是window的属性
a=10; —–>window.a=10;
var b=534; —–>window.b=534;
window就是全局的域
function test(){
var a=b=123;
}
test();
连续赋值从右向左赋值，相当于把123赋给b，b再赋给a，但此时就相当于是未声明b就直接赋值给b，b归window所有
console.log(window.a) —-> undefined （a不是全局变量）
console.log(window.b) —–> 123

预编译
预编译发生在函数执行的前一刻，解决函数执行顺序
function fn(a){
console.log(a);
var a=123;
console.log(a);
function a() {}
console.log(a);
var b=function(){}
console.log(b);
function d(){}
}
fn(1)
1.创建AO对象 Activation Object 执行期上下文
相当于是函数产生的存储空间库

2.找形参和变量声明，将变量和形参名作为AO对象的属性名
值统一是undefined

AO{
a:undefined,
b:undefined，
}
3.将实参和形参相统一

AO{
a:1,
b:undefined，
}
4.在函数体里面找函数声明作为AO对象的属性名，值赋予函数体

AO{
a:1,
b:undefined，
d:function d()
}
此方法解决了优先级顺序，谁覆盖谁的问题
